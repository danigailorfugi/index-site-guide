# Index Site Guide

## 项目简介

本项目是一个用于归档和发布多个独立 HTML 页面的仓库。这些页面可能包含信息展示、工具页面、静态演示等内容，彼此之间没有明确的域名或网站关联。仓库的主要目的是提供一个统一的位置，用于存放和分发这些独立的 HTML 文件。

## 目录结构

```
/
├── pages/          # 存放所有独立的 HTML 页面文件
├── assets/         # 页面引用的公共资源（如 CSS、JS、图片等）
├── index.html      # 可选的主页索引文件
└── README.md       # 本文件
```

## 页面归档说明

- 每个 HTML 页面应放置在 `pages/` 目录下，文件名需唯一且具有描述性。
- 页面应尽量保持自包含，若依赖外部资源，建议将资源文件放入 `assets/` 目录，并在页面中引用相对路径。
- 归档的页面不限制具体用途，但应避免包含恶意代码、侵权内容或违反平台服务条款的材料。
- 页面内容可能不定期更新或移除，恕不另行通知。

## 维护说明

- 本仓库由维护者不定期更新，不接受外部直接提交的页面文件。
- 若发现页面存在显示异常、链接失效或安全问题，欢迎通过 Issues 反馈。
- 对于大型或频繁更新的页面归档需求，建议另行创建独立仓库。
- 本仓库不提供任何形式的托管保证，页面可用性取决于 GitHub Pages 或类似服务的稳定性。

## 其他

- 本仓库的 HTML 页面仅供学习、参考或个人使用，不构成任何形式的服务承诺。
- 使用本仓库中的任何页面时，请遵守相关法律法规及 GitHub 服务条款。
